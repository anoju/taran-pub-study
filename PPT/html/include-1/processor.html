Part4-1CSS Pre-processor

CSS Pre-processorCSS 전처리기
자신만의 특별한 구문(Syntax)을 가지고 CSS를 생성하는 프로그램입니다. CSS의 문제점을 프로그래밍 방식, 즉 변수·함수·상속 등 일반적인 프로그래밍 개념을 사용하여 보완합니다. CSS 전처리기에는 다양한 모듈이 존재하는데 그 중에서도 Sass·Less·Stylus가 가장 많이 사용됩니다.

CSS 전처리기는 공통 요소 또는 반복적인 항목을 변수 또는 함수로 대체할 수 있는 재사용성, 임의 함수 및 내장 함수로 인해 개발 시간과 비용 절약, 중첩·상속과 같은 요소로 인해 구조화된 코드 유지 및 관리 용이 등의 장점이 있습니다. 반면 전처리기를 위한 도구가 필요하고 다시 컴파일하는데 시간이 소요된다는 단점도 존재합니다.

Part4-2CSS Pre-processor 종류

  1. 1. Sass (Syntactically Awesome Style Sheets)

    • Sass는 가장 널리 사용되는 CSS 전처리기입니다. Sass는 Ruby 언어로 작성되어 있으며,
    • 문법은 Sass(들여쓰기)와 Scss(중괄호) 두 가지로 나뉘어 있습니다.

style.sass


                $main-color: #d00
                @mixin pad($x, $y) 
                  padding: $y $x
                nav
                  ul
                    display: flex
                  @for $i from 1 through 3
                    li:nth-child(#{$i})
                      margin-left: $i * 5px
                  a
                    display: block
                    @include pad(5px, 10px)
                    color: $main-color

style.scss


                $main-color: #d00;
                @mixin pad($x, $y) {
                  padding: $y $x;
                }
                nav {
                  ul {
                    display: flex;
                  }
                  @for $i from 1 through 3 {
                    li:nth-child(#{$i}) {
                      margin-left: $i * 5px;
                    }
                  }
                  a {
                    display: block;
                    @include pad(5px, 10px);
                    color: $main-color;
                  }
                }

style.css


            nav ul{display:-webkit-box;display:-ms-flexbox;display:flex}nav li:nth-child(1){margin-left:5px}nav li:nth-child(2){margin-left:10px}nav li:nth-child(3){margin-left:15px}nav a{display:block;padding:10px 5px;color:#d00}

  1. 2. Less (Leaner CSS)

    • Less는 JavaScript를 기반으로한 CSS 전처리기입니다.
    • 문법은 Scss와 유사하지만, 변수나 믹스인 등을 정의할 때 @를 사용하는 등의 문법적 차이가 있습니다.

style.less


                @main-color: #d00;
                .pad(@x, @y) {
                  padding: @y @x;
                }
                nav {
                  ul {
                    display: flex;
                  }
                  each(range(3), {
                    li:nth-child(@{value}) {
                      margin-left: (@value * 5px);
                    }
                  });
                  a {
                    display: block;
                    .pad(5px, 10px);
                    color: @main-color;
                  }
                }

style.css


            nav ul{display:-webkit-box;display:-ms-flexbox;display:flex}nav li:nth-child(1){margin-left:5px}nav li:nth-child(2){margin-left:10px}nav li:nth-child(3){margin-left:15px}nav a{display:block;padding:10px 5px;color:#d00}

왜 Sass는 종류가 2가지(Sass와 Scss) 일까요?
Sass(Syntactically Awesome Style Sheets)는 원래 Ruby 기반으로 작성된 CSS 전처리기로, 인기를 얻기 시작했습니다. 그러나 Sass의 문법은 CSS와는 매우 다르기 때문에 Sass를 처음 사용하는 개발자들에게는 학습 곡선이 꽤 높을 수 있습니다. 이러한 이유로 Sass의 개발자는 Sass의 문법이 CSS와 유사하도록 Sass의 새로운 버전을 만들기로 결정하였습니다. 이로 인해 Scss(Sassy CSS)가 개발되었습니다.

Scss는 CSS와 거의 동일한 구문을 사용하며, 중괄호({})와 세미콜론(;)을 사용하여 속성을 정의합니다. 이전 버전의 Sass와 비교하면 Scss는 보다 간단하고 직관적인 구문을 사용하므로 CSS에 익숙한 개발자도 쉽게 학습할 수 있습니다.
이제 Sass는 알겠는데 발음이 비슷한 SaaS는 무엇이죠?
SaaS는 Software as a Service의 약어로, 소프트웨어 서비스를 의미합니다. 즉, 소프트웨어를 구매하거나 설치하지 않고 인터넷을 통해 이용할 수 있는 클라우드 컴퓨팅 모델입니다.
SaaS는 일반적으로 인터넷 브라우저를 통해 접근할 수 있는 웹 기반의 소프트웨어로 제공됩니다. 이러한 소프트웨어는 일반적으로 구독 기반으로 제공되며, 사용자는 구독 기간 동안 서비스를 이용할 수 있습니다.
SaaS의 예로는 Google Docs, Microsoft Office 365와 같은 애플리케이션 서비스 제공업체(ASP)와 인사 관리 소프트웨어, 전자 상거래 시스템, 고객 관계 관리 툴, 통합 개발 환경(IDE)을 제공하는 엔터프라이즈 서비스가 있습니다.

Part4-3CSS Post-processor

CSS Post-processorCSS 후처리기
CSS 후처리기의 가장 대표적인 언어는 PostCSS입니다. JS 플러그인을 사용하여 CSS를 변환하는 도구로 CSS의 Babel로 볼수 있습니다.

장점은 모쥴식 사용으로 필요한 플러그인만 포함하여 사용할수 있고, 전체를 다 포함하지 않아도 되므로 속도가 빠릅니다. JavaScript 기반으로 플러그인을 직접 개발 할 수도 있으며 CSS표준문법을 기반으로 하기때문에 별도의 학습이 필요가 없습니다.
단점은 전처리기에 비해 강력한 문법은 제공하지 않습니다. 필요한 플러그인은 직접 찾아서 조립해야하며 사용하는 플러그인 지원이 중단 될 수도 있습니다.

Part4-7Scss 추천하는 이유

CSS 전/후처리기를 사용하려면 각각 언어마다
Grunt, gulp, webpack등등 을 이용하여 전처리기를 사용할 수 있도록 세팅하는 과정이 필요했는데
처음 접하는 사람에게는 그 준비과정은 어려울 수 있습니다.

지금은 개발툴의 발전으로 ( Visual Studio Code[1]에서 1개의 확장프로그램(Live Sass Compiler)만 설치 후 실행만 하면) 자동으로 css 파일로 컴파일되며, 실행 후 전처리기 파일을 수정 및 작성을 할 때 마다 실시간으로 CSS 파일로 컴파일 됩니다.
Live server[2] 와 같이 사용하면 html 문서 작성 시 아주 편리하게 가능합니다.

LESS나 Stylus도 지원하는 확장 프로그램이 있긴하지만 Sass 관련 확장프로그램이 우수하며,
Scss가 다른 언어에 비해 유저풀이 넓고, 학습 곡선이 낮기때문에 좀 더 추천하고 있습니다.

[1]: Visual Studio Code : 마이크로 소프트사에서 무료 라이센스로 제공하며, 현재 개발자들이 제일 많이 사용하는 에디터(개발툴)

[2]: html 수정 및 작성 시 실시간으로 브라우저에서 자동 새로 고침되어 해당 내용을 보여주는 vscode 확장프로그램

Part4-8CSS framework의 종류

CSS 프레임워크(CSS framework)는 웹 개발에서 자주 사용되는 CSS 속성들을 미리 정의하여 일괄적으로 적용할 수 있도록 도와주는 도구입니다. 이를 통해 빠른 개발이 가능해지며, 일관된 디자인을 구현할 수 있습니다.

  1. 1. Bootstrap (2011)

    • 장점: 가장 인기 있는 CSS 프레임워크 중 하나로서, 대부분의 브라우저에서 일관성 있는 디자인을 제공. 다양한 컴포넌트와 유틸리티 클래스를 제공하여 빠르고 쉽게 웹사이트를 개발. 커스터마이징이 쉽고, 높은 수준의 문서화와 활발한 커뮤니티를 가지고 있음
    • 단점: Bootstrap을 사용하여 개발된 웹사이트는 구별하기 어려울 수 있음. 과도한 사용으로 인해 특정 웹사이트에 대한 고유한 디자인 요소를 잃을 수 있음. 일부 컴포넌트가 다른 CSS 프레임워크에 비해 무거움

  2. 2. Foundation (2011)

    • 장점: 반응형 디자인을 위한 그리드 시스템과 다양한 컴포넌트를 제공. 컴포넌트를 구성하는 다양한 요소들을 자유롭게 조합하여 커스텀 컴포넌트를 만들 수 있음. 다양한 브라우저와 호환되며, 문서화와 커뮤니티 활동이 활발
    • 단점: 컴포넌트의 다양성이 다른 CSS 프레임워크에 비해 적을 수 있음. 최신 기술과 디자인 트렌드를 따라가지 않는 것처럼 보일 수 있음. 러닝 커브가 다른 CSS 프레임워크보다 높음

  3. 3. Materialize (2014)

    • 장점: Google의 Material Design 스펙을 기반으로 제작되었기 때문에, 모던하면서도 적절한 디자인을 제공. 반응형 디자인을 위한 그리드 시스템과 다양한 UI 컴포넌트를 제공. 레이아웃과 스타일이 일관성 있게 유지
    • 단점: 일부 개발자들에게는 Material Design 스펙에 따른 디자인이 마음에 들지 않을 수 있음. 컴포넌트의 수가 다른 CSS 프레임워크에 비해 적음. 커스터마이징하기에는 다소 제한적

Part4-8CSS framework의 종류

  1. 4. Bulma (2016)

    • 장점: 높은 수준의 커스터마이징이 가능. 간결하고 직관적인 클래스 이름을 사용하여 개발자가 빠르게 스타일을 적용. 다양한 레이아웃을 지원하며, 반응형 웹 개발이 용이
    • 단점: 컴포넌트의 다양성이 다른 CSS 프레임워크에 비해 적음. 특정 기능이 부족. 문서화 및 커뮤니티 활동이 다른 CSS 프레임워크에 비해 떨어짐

  2. 5. Semantic UI (2014)

    • 장점: 시맨틱 마크업에 대한 지원이 매우 강함. 모듈화된 구조로 구성되어 있어 필요한 컴포넌트만 쉽게 사용. 컴포넌트를 사용하여 반응형 웹사이트를 빠르고 쉽게 개발
    • 단점: 커스터마이징이 어려움. 다른 CSS 프레임워크에 비해 속도가 느림. 문서화가 부족하며 커뮤니티의 활동도 다른 CSS 프레임워크에 비해 상대적으로 적음

  3. 6. Tailwind CSS (2017)

    • 장점: 매우 유연하고 커스터마이징이 쉬움. 클래스 기반으로 구성되어 있어 HTML 마크업에 직접 스타일을 적용할 필요가 없음. 다양한 디자인 요소와 유틸리티 클래스를 제공하여 빠르고 쉽게 웹사이트를 개발
    • 단점: 초기 학습 곡선이 높고, 클래스 기반의 스타일링 방식이 익숙하지 않은 개발자들에게는 부담스러울 수 있음. 스타일을 클래스 기반으로 적용하는 것이 일관성 있는 디자인을 얻는 것보다 복잡할 수 있음

  4. 7. PureCSS (2013)

    • 장점: 매우 가볍고 빠름. 커스터마이징이 용이하며, 필요한 부분만 사용가능. CSS 코드가 간결하며, 가독성이 뛰어남.
    • 단점: 제공되는 컴포넌트의 수가 적고, 디자인이 매우 간단하며, 모던한 디자인 요소가 부족. 그리드 시스템이 다른 CSS 프레임워크에 비해 제한적.

Part4-8CSS framework의 종류

  1. 8. Tachyons (2014)

    • 장점: 클래스 기반 시스템으로 코드의 재사용성이 높음. CSS 파일이 매우 작아 빠르게 로드. 높은 유연성을 제공하며, 커스터마이징이 용이
    • 단점: 클래스 이름이 의미를 파악하기 어렵고, 기본 디자인이 단순하며, 모던한 디자인 요소가 부족.그리드 시스템이 다른 CSS 프레임워크에 비해 제한적.

  2. 9. Ant Design (2016)

    • 장점: 디자인 시스템과 UI 컴포넌트를 제공. React, Vue, Angular 등의 프레임워크와 함께 사용하기 적합. 컴포넌트와 디자인의 일관성이 높아, 개발자와 디자이너 간의 협업이 용이.
    • 단점: 대부분의 컴포넌트가 중국어로 되어있어, 영어권 개발자에게는 사용이 불편. 무겁고 큰 크기의 CSS 파일을 제공하기 때문에, 로드 속도가 느림. 커스터마이징이 어려움

  3. 10. UIKit (2013)

    • 장점: 반응형 웹을 쉽게 개발. 컴포넌트의 일관성이 높아, 디자인이 일관된 웹 페이지를 만듬. CSS와 JavaScript가 포함된 것으로 모든 필수 기능을 제공.
    • 단점: 컴포넌트가 다른 CSS 프레임워크에 비해 적음. 디자인이 다소 구식임. 크기가 큰 CSS 파일을 포함하고 있어 로딩 속도가 느림

  1. 11. Primer (2021)

    • 장점: GitHub에서 개발하고 관리하고 있어, GitHub 페이지 개발에 용이. 유용한 컴포넌트와 모듈을 제공하며, 일관된 디자인을 유지. 커스터마이징이 쉽고 유연
    • 단점: 컴포넌트 수가 다른 CSS 프레임워크에 비해 적음. 디자인이 매우 간단하고 기본적이며, 모던한 디자인 요소가 부족. 그리드 시스템이 다른 CSS 프레임워크에 비해 제한적

  2. 12. Halfmoon (2021)

    • 장점: 반응형 웹사이트를 빠르고 쉽게 개발할 수 있음. 유용한 컴포넌트와 유틸리티 클래스를 제공하며, 개발자가 자유롭게 커스터마이징. 모든 컴포넌트에 대한 문서화와 예제 코드가 제공
    • 단점: 컴포넌트 수가 다른 CSS 프레임워크에 비해 적음. 최신 기술과 디자인 트렌드를 따라가지 않는 것처럼 보일 수 있음. 기능이나 커스터마이징에 대한 기능이 다른 CSS 프레임워크에 비해 제한적
