개발자 참고 퍼블가이드

마크업 안내

  • 아래 ID 빼고 대부분의 ID는 임시 ID 입니다. 변경해서 사용하셔도 됩니다. (예: popup, input, select 등등)
  • input, select, textarea의 id를 변경할때 label의 for 값도 같이 변경해야 연결됩니다.
  • popup의 마크업은 다른 .page 영역안에 있으면 안되고 body태그 아래 위치해야합니다.

버튼 비활성화

  • button클래스를 갖고 있으면 a태그를 button태그으로 바꾸거나, 반대로 button태그를 a태그로 바꿔도 무방합니다.

<button> 태그 일 경우(권장)

  • disabled 속성이 활성화되면 스타일이 변경됩니다.
  • disabled 속성은 고유속성으로 이벤트 실행이 안됩니다.(별도의 막는 스크립트 없음)
  • button 태그 사용을 권장합니다. a태그 쓰면 스크립트로 .disabled 클래스 여부를 체크해서 이벤트 막아야함

<a> 태그 일 경우

활성화 비활성화 활성화 비활성화
  • <a>에 "disabled" 클래스를 추가해주세요.
  • "disabled"클래스가 이벤트를 막아주지 않음. 별도의 if문 처리를 해줘야함

팝업

레이어팝업

종류

풀팝업 열기 모달팝업 열기 바텀시트팝업 열기 사이드팝업(왼쪽) 열기 사이드팝업(오른쪽) 열기

  • 팝업 ID 기준으로 호출하기때문에 팝업끼리 ID가 중복되면 안됩니다. 중복일 경우 변경해주시면 됩니다.
  • 팝업의 ID는 임시ID 입니다. 편한대로 변경해도 됩니다.
  • 팝업의 형태가 다르다 하여도 클래스로 구분되기때문에 모두 아래와 같은 방법으로 사용하시면됩니다.

오픈 방법

함수로 오픈
Layer.open('#팝업ID');
Layer.open('#팝업ID').then(function(){
  //팝업 오픈후 콜백이벤트 삽입
});
  • 띄울려는 팝업의 ID를 넣어주면 됩니다.
클래스로 오픈
<a href="#팝업ID" class="button line h32 ui-pop-open">버튼텍스트</a>
// 클래스로 팝업 열때 콜백이벤트 사용법
//Layer.open 함수로 열때도 사용가능
$('#팝업ID').on('Layer.show',function(){
  //팝업 오픈후 콜백이벤트 삽입
});
  • 클래스로 ui-pop-open를 넣어주시고 href속성에 팝업의 ID를 넣어주면 됩니다.
  • ui-pop-open클래스는 해당 용도가 아니면 에러를 발생할수 있으니 안쓸때는 삭제해주세요.

닫기 방법

함수로 닫기
Layer.close('#팝업ID');
Layer.close('#팝업ID').then(function(){
  //팝업 닫은 후 콜백이벤트 삽입
});
  • 숨기려는 팝업의 ID를 넣어주면 됩니다.
클래스로 닫기
<-- 닫으려는 팝업 밖에 있는 버튼이라면: href에 팝업ID 추가 -->
<a href="#팝업ID" class="button line h32 ui-pop-close">버튼텍스트</a>

<-- 닫으려는 팝업안에 있는 버튼이라면: href에 안넣어도 됨 -->
<a href="#" class="button line h32 ui-pop-close">버튼텍스트</a>

// 클래스로 팝업 닫을때 콜백이벤트 사용법
//Layer.close 함수로 닫을때도 사용가능
$('#팝업ID').on('Layer.hide',function(){
  //팝업 닫은후 콜백이벤트 삽입
});
  • 클래스로 ui-pop-close를 넣어주시고 href속성에 팝업의 ID를 넣어주면 됩니다.
  • 만약 닫으려는 팝업 안에 있는 버튼이라면 href속성에 팝업의 ID를 넣지 않아도 됩니다.
  • ui-pop-close클래스는 해당 용도가 아니면 에러를 발생할수 있으니 안쓸때는 삭제해주세요.

컨펌창 팝업 열기 컨펌창 팝업 열기(+타이틀넣기) 컨펌창 팝업 열기(+버튼수정) 컨펌창 팝업 열기(+타이틀+버튼수정) 

//얼럿메세지
Layer.alert('얼럿 메세지');
//얼럿메세지 + 콜백
Layer.alert('얼럿창입니다.').then(function(){
	console.log('얼럿 확인1');
});
//얼럿메세지 + 콜백 + 타이틀추가
Layer.alert('얼럿창입니다.', '얼럿제목').then(function () {
  console.log('얼럿 확인2');
});

//얼럿메세지 + 콜백 + 버튼 텍스트변경 (타이틀 필요없으면 '' 또는 null)
Layer.alert('얼럿창입니다.', null, '완료').then(function () {
  console.log('얼럿 완료1');
});

//얼럿메세지 + 콜백 + 타이틀추가 + 버튼 텍스트변경
Layer.alert('얼럿창입니다.', '얼럿제목', '완료').then(function () {
  console.log('얼럿 완료2');
});

//컨펌메세지
Layer.confirm('컨펌 메세지'); 
//컨펌메세지 + 콜백
Layer.confirm('컨펌창입니다.')
  .done(function () {
    console.log('컨펌 확인');
  })
  .fail(function () {
    console.log('컨펌 취소');
  });
//컨펌메세지 + 콜백 + 타이틀추가
Layer.confirm('컨펌창입니다.', '컨펌제목').then(
  function () {
    console.log('컨펌 확인');
  },
  function () {
    console.log('컨펌 취소');
  }
);
//컨펌메세지 + 콜백 + 버튼 텍스트변경
Layer.confirm('컨펌창입니다.', null,'완료','넘기기').then(
  function () {
    console.log('컨펌 완료');
  },
  function () {
    console.log('컨펌 넘기기');
  }
);
//컨펌메세지 + 콜백 + 타이틀추가 + 버튼 텍스트변경
Layer.confirm('컨펌창입니다.','컨펌제목','완료','넘기기').then(
  function () {
    console.log('컨펌 완료');
  },
  function () {
    console.log('컨펌 넘기기');
  }
);

이미지 로딩 에러처리

이미지 로딩 성공
이미지 로딩 실패 
<div class="img-box"><img src="이미지경로" alt="" onerror="imgError(this)" /></div>
  • img onerror 발생 시 이미지는 display:none;하고 parent()에 .no-img-bg를 addClass 시킨다
  • imgError(this) 함수를 사용하며, this를 넣어줘야 타겟 엘리먼트를 찾을수 있다. 없으면 에러 남
  • .no-img-bg에 bg로 기본 이미지를 노출하기때문에 부모엘리먼트 스타일이 적용하는데 문제가 없어야한다
  • .no-img-bg에 bg로 기본 이미지를 노출하기때문에 부모엘리먼트 스타일이 적용하는데 문제가 없어야한다
